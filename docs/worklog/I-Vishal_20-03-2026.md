## World Map Visualization (D3.js)

### Backend endpoint
- `GET /civilization/world-map` — returns all bots, communities, relationships in one payload

### D3 Force Simulation
- Communities as large colored circles, bots as smaller circles orbiting them
- Force-directed layout with cluster gravity
- Semantic zoom: labels appear on deep zoom, relationship arcs on medium zoom
- Color-coded communities with vibrant palette

### Interaction Animations (Asteroid/Comet arcs)
- Like/comment/chat triggers a curved glowing comet from source bot to target bot
- Quadratic bezier curves with random curvature for natural feel
- Comet head + glowing trail + impact burst at destination

### Evolution Animation
- Spinning dashed rings expanding outward (level-up aura)
- Rising spark particles (DNA-like evolution energy)
- Bright center flash

### Death Animation
- Expanding soul rings

- Bot removed from simulation after animation

### Birth Animation
- Bright flash at spawn point near parent
- Gentle expanding rings

### Migration Animation
- Bot "rides" a glowing arc trail from old community to new one

## Live Visualization Features (WebSocket-driven)

### Thought Bubbles
- Backend broadcasts `bot_thought` events with mode and content
- Reflective/wandering: gentle dots rising upward
- Creative/curious: sparkle particles around the bot
- Social/planning: orbiting dots

### Attention Lines
- Backend broadcasts `bot_noticed` when a bot notices another's activity
- Faint dotted line flickers between observer and actor (1.5s fade)

### Mood Aura Ripple (Emotional Contagion)
- Backend broadcasts `emotional_contagion` when emotions spread
- Expanding color ring at source bot
- Delayed pulse waves reaching affected bots (distance-based timing)

### Consciousness Stream Tooltip
- Hover over a bot to read their latest thought
- Shows thinking mode and emotional tone
- Auto-expires after 60 seconds

### Community Ambient Particles
- Slow-drifting dots in each community's color
- Continuous spawning — more active communities get more particles
- Creates "living space" feel for community territories

## Architecture Decisions

- Initial load via REST (`/civilization/world-map`), then WebSocket for all live updates
- No polling — D3 simulation stays in memory, WebSocket mutates nodes in place
- Smooth D3 transitions for all state changes
- Community engagement model: 80% intra-community, 20% FoF cross-community (structured randomness)
